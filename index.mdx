Confluence Markdown Exporter (`cme`) is a command-line tool that converts Confluence pages into clean Markdown files using the Atlassian REST API. Whether you're migrating content to Obsidian, Azure DevOps Wikis, or any Markdown-based platform, `cme` handles the conversion automatically — including headings, tables, code blocks, images, diagrams, and more.

## What you can export

`cme` supports four export scopes, letting you target exactly what you need:

## Key features

- **Incremental exports** — only re-exports pages that changed since the last run, making repeated exports fast
- **Parallel processing** — exports multiple pages simultaneously with configurable worker count
- **Rich element support** — converts tables, code blocks, tasks, alerts, images, draw.io diagrams, and PlantUML
- **Jira enrichment** — fetches Jira issue data to enrich linked issues with their summaries
- **Target-system presets** — ready-made configurations for Obsidian, Azure DevOps Wikis, and more
- **CI-friendly** — clean log output, environment variable overrides, and non-interactive mode

## Get started

<Steps>
  <Step title="Install cme">
    Install the exporter using the one-line installer for your platform. See [Installation](/installation) for all options.

    ```bash
    curl -LsSf uvx.sh/confluence-markdown-exporter/install.sh | sh
    ```
  </Step>
  <Step title="Configure your credentials">
    Run the interactive configuration menu to add your Confluence URL and API token:

    ```bash
    cme config edit auth.confluence
    ```
  </Step>
  <Step title="Set your output path">
    Choose where exported Markdown files will be saved:

    ```bash
    cme config set export.output_path=./confluence-export
    ```
  </Step>
  <Step title="Run your first export">
    Export a Confluence space or individual page:

    ```bash
    cme spaces https://company.atlassian.net/wiki/spaces/MYSPACE
    ```
  </Step>
</Steps>
