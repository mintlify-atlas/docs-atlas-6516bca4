---
title: "Quickstart: run your first Confluence Markdown export"
description: "Install cme, configure Confluence credentials, set your output path, and run your first space or page export from start to finish in minutes."
---

This guide walks you through installing `cme`, connecting it to your Confluence instance, and exporting your first space or page. The whole process takes a few minutes.

<Steps>
  <Step title="Install cme">
    Install `cme` using the one-line installer for your platform:

    ```bash
    curl -LsSf uvx.sh/confluence-markdown-exporter/install.sh | sh
    ```

    For Windows or alternative install methods, see the [Installation](/installation) page.
  </Step>

  <Step title="Configure Confluence credentials">
    Run the interactive editor to add your Confluence URL, username, and API token:

    ```bash
    cme config edit auth.confluence
    ```

    The interactive prompt will ask for:

    - **URL** — your Confluence base URL, e.g. `https://company.atlassian.net`
    - **Username** — your Atlassian account email address
    - **API token** — an API token from your Atlassian account (see below)

    <Note>
      Always use `cme config edit auth.confluence` to set credentials — not `cme config set`. Auth keys are indexed by URL, so the interactive editor handles them correctly. Using `cme config set` directly for auth keys will not work as expected.
    </Note>

    **Generating an API token**

    For Confluence Cloud, generate a token at [id.atlassian.com/manage-profile/security/api-tokens](https://id.atlassian.com/manage-profile/security/api-tokens):

    1. Sign in to your Atlassian account.
    2. Click **Create API token**.
    3. Give it a name (e.g. `cme`) and copy the token.
    4. Paste it into the `cme config edit auth.confluence` prompt.

    For scoped (fine-grained) API tokens, grant the following classic scopes for full read-only access:

    ```text
    read:confluence-content.all
    read:account
    read:confluence-content.permission
    read:confluence-content.summary
    read:confluence-groups
    read:confluence-props
    read:confluence-space.summary
    read:confluence-user
    read:me
    readonly:content.attachment:confluence
    search:confluence
    ```

    For Confluence Server, use a Personal Access Token (PAT) instead of an API token. The interactive editor will prompt for the correct fields based on your instance type.
  </Step>

  <Step title="Set your output path">
    Tell `cme` where to write the exported Markdown files:

    ```bash
    cme config set export.output_path=./output
    ```

    You can use any absolute or relative path. The directory will be created automatically if it does not exist.
  </Step>

  <Step title="Export a space">
    Run your first export by passing a Confluence space URL:

    ```bash
    cme spaces https://company.atlassian.net/wiki/spaces/MYSPACE
    ```

    Supported space URL formats:

    | Type | Example |
    |---|---|
    | Cloud | `https://company.atlassian.net/wiki/spaces/SPACEKEY` |
    | Server (long) | `https://wiki.company.com/display/SPACEKEY` |
    | Server (short) | `https://wiki.company.com/SPACEKEY` |

    You can also export individual pages or page trees:

    ```bash
    # Export a single page
    cme pages https://company.atlassian.net/wiki/spaces/KEY/pages/123456789/Page+Title

    # Export a page and all its descendants
    cme pages-with-descendants https://company.atlassian.net/wiki/spaces/KEY/pages/123456789/Root

    # Export everything across all spaces in your org
    cme orgs https://company.atlassian.net
    ```
  </Step>

  <Step title="Inspect the output">
    After the export completes, your output directory contains one folder per space. Pages are nested to mirror the Confluence page hierarchy:

    ```text
    output/
    └── MYSPACE/
       ├── MYSPACE.md
       └── MYSPACE/
          ├── My Confluence Page.md
          └── My Confluence Page/
                ├── My nested Confluence Page.md
                └── Another one.md
    ```

    Each `.md` file contains the converted Markdown content. Attachments are saved under `MYSPACE/attachments/` by default.

    On subsequent runs, `cme` skips pages that have not changed — only modified pages are re-exported.
  </Step>
</Steps>

<Tip>
  Run `cme --help` at any time to see all available commands, or append `--help` to any subcommand for detailed usage:

  ```bash
  cme spaces --help
  cme config --help
  ```
</Tip>
